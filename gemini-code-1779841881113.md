# UX & UI 规范文档: Do You Want to Accept Cookies?

**项目类型:** 互动式网页艺术 / 高保真前端原型
**核心体验基调:** 温暖复古水彩 vs. 冰冷数据监控 (Sweet Surveillance)
**目标视口:** MacBook 桌面端全屏 (比例 16:10)，采用 `object-fit: contain` (Letterboxing) 缩放策略。
**交互定位:** 纯鼠标/触控板驱动的互动装置（豁免标准 WCAG 键盘与屏幕阅读器无障碍要求）。

---

## 1. 信息架构 (Information Architecture)

### 1.1 页面层级与状态流转
整个应用为单页应用 (SPA) 架构，通过前端路由或状态机在 6 个核心视图（Views）间进行无刷新切换。

### 1.2 布局区域与视图容器分配
* **安全渲染区 (Safe Area):** 设定基础画布容器比例为 `16:10`（例如基准分辨率设为 `1920x1200` 或 `1440x900`）。
* **视口适配 (Viewport Scaling):** 外层容器使用 CSS 居中并应用缩放变换（Scale Transform）或 `aspect-ratio` 控制。当用户浏览器窗口非 16:10 时，容器在垂直或水平方向居中，周围留出底色黑边/奶油色边框，确保内部绝对定位的坐标 100% 不变形。
* **Z-Index 层级规划:**
    * `z-index: 10` - 背景层 (瓷砖/桌面图片, 底层视频播放器)
    * `z-index: 20` - 内容层 (食材实体, 饼干罐, 按钮, 碗)
    * `z-index: 30` - 交互层 (Hover 弹窗, 拖拽中的食材)
    * `z-index: 40` - 突变/中断层 (微阻断警告纸条, Glitch 故障代码弹窗)
    * `z-index: 50` - 鼠标样式层 (自定义复古光标/搅拌器)

## 2. 核心用户流程 (Core User Flows)

以下为前端状态流转逻辑，包含了关键的微阻断分支：

- **Page 1: 封面/入口**
  - Click 'Let's Bake One' -> 跳转至 Page 2
- **Page 2: 散落的桌面场景**
  - **交互分支 A：** 悬停并查看食材背后的数据含义 -> 正常选择并拖入 4 个食材 -> 跳转至 Page 3
  - **交互分支 B：** 未悬停直接快速连点/拖拽 4 个 -> 触发微阻断 (弹出复古警告纸条) -> 点击确认/继续 -> 跳转至 Page 3
- **Page 3: Mixing 搅拌**
  - 监听鼠标圆周运动轨迹与速度 -> 达到搅拌完成阈值 -> 自动流转至 Page 4
- **Page 4: Baking 烘焙动画**
  - 强制播放固定时长 5-8s -> 自动流转至 Page 5
- **Page 5: Reveal 最终结果**
  - 打分系统结算，播放对应破坏性视频并展示揭晓文案
  - Click 'Bake Another Cookie' -> 跳转至 Page 6
- **Page 6: 铁皮盒画廊**
  - Click 'Try a Safer Batch' -> 重置状态并返回 Page 2

## 3. 视图规范 (View Specifications)

### 3.1 Page 2: Ingredients (食材选择)
* **背景:** 极具生活化的复古木纹/餐布桌面（全屏背景图）。
* **布局逻辑 (非网格):** 放弃 Flexbox/Grid，前端需使用**绝对定位 (Absolute Positioning)** 通过百分比 `top` 和 `left` 属性，将 12 个食材不规则地“散落”在画布的左侧和下方区域。碗（Dropzone）固定放置在右侧区域。
* **组件层级:**
    * `Ingredient Node`: 包含默认的水彩图片。
    * `Tooltip Node` (默认 `opacity: 0`): 绑定在食材上方，Hover 时渐显，展示复古标签样式的“真实数据含义”。

### 3.2 Page 3: Mixing (搅拌失控)
* **视图转换:** 桌面的食材消失，视角拉近（Zoom-in），碗占据画面中央。
* **动态文本层:** 碗的上方叠加一层绝对定位的文本容器（`pointer-events: none`），用于在用户搅拌过快时，通过 JavaScript 控制随机闪现法律条款（如 "consent to third-party access"），并附带 CSS `filter: blur()` 和扭曲效果。

### 3.3 Page 4 & 5: Video & Baked-in Text (音视频渲染层)
* **架构变更:** 这两页的视觉核心交由预渲染的视听资产（由 After Effects 和 Audition 导出的 `.mp4` 或 `.webm` 文件）接管。
* **Page 5 的 DOM 结构:**
    * **背景:** HTML5 `<video>` 标签全屏覆盖，播放由“打分系统”算出的 6 种结果之一。由于揭晓的文字（Contains / Shaped by）已直接烘焙（Baked-in）到视频流中，前端只需确保视频的无缝播放和循环。
    * **叠加层:** 前端仅需在视频尾声阶段，叠加一个透明的“Bake Another Cookie”操作按钮即可。

### 3.4 Page 6: Replay (复古铁皮饼干盒画廊)
* **容器隐喻:** 背景图为一张打开的精致复古铁皮饼干盒。
* **槽位布局 (Slot Coordinates):** 盒内有 6 个特定形状的凹槽。前端维护一个包含 6 个绝对定位坐标的数组。
* **状态渲染 (State Rendering):**
    * **空状态 (Locked):** 读取本地内存，未解锁的 Cookie 渲染为一张暗色的垫纸或水彩剪影图层。
    * **已解锁 (Unlocked):** 渲染对应 Cookie 的精美切片图层，附带微弱的 `drop-shadow` 以增加立体感。

## 4. 交互模式 (Interaction Patterns)

### 4.1 微阻断判定 (Micro-friction UI)
* **事件监听:** 维护一个全局变量 `hasHoveredTerms = false`。监听所有食材的 `mouseenter` 事件，只要触发一次且停留超过 0.5s，变量设为 `true`。
* **触发条件:** 当碗中被放入第 4 个食材时检查该变量，如果为 `false`，则暂停页面跳转，在屏幕中央弹出 `z-index: 40` 的复古纸条弹窗：“Are you sure you don't want to read the labels?”。

### 4.2 物理级别的拖拽体验 (Drag & Drop)
* 使用前端物理库（如 `framer-motion` 或纯原生 JS 的 `PointerEvents`）实现食材的拖放。
* 当食材被拾起（`pointerdown`）时，略微放大 (`scale: 1.05`) 并倾斜，增加真实的抓取感。
* 碗作为 Dropzone，拖入时应有高亮或物理碰撞的微反馈。

### 4.3 鼠标速率映射 (Speed Mapping)
* **圆周监听:** 在 Page 3，通过 `requestAnimationFrame` 追踪鼠标的 X/Y 坐标，计算角速度（Angular velocity）。
* **映射输出:** 将角速度值映射为 CSS 变量 `--blur-amount` 和 HTML5 Audio API 中背景音效的 `playbackRate` 或滤波器参数。搅拌越快，画面越模糊，音频越刺耳。

### 4.4 故障突变 (Glitch Transition)
* 在 Page 5 视频播放初期或特定关键帧，前端可以通过短暂切换背景 `<canvas>` 或使用 CSS 混合模式（如 `mix-blend-mode: exclusion`），配合冰冷的系统默认字体（Monospace）代码弹窗，瞬间击碎水彩复古感。

## 5. 设计系统集成 (Design System Integration)

### 5.1 色彩调色板 (Color Palette)
* **Seduction 层 (诱导与包装):** 奶油白 (`#FDFBF7`)，柔和棕 (`#8C6D58`)，粉糖色 (`#F5C5C9`)，香草黄 (`#F4E7C3`)。
* **Exposure/Control 层 (真实数据系统):** 终端黑 (`#1A1A1A`)，荧光绿 (`#39FF14`)，报错红 (`#FF3333`)，冷峻蓝 (`#0055FF`)——*仅在 Glitch 和 Page 5 揭示时突兀出现*。

### 5.2 排版与字体 (Typography)
* **表象字体 (The Façade):** 优雅的复古衬线体或手写体（如 *Playfair Display* 或 *Caveat*），用于标题、包装标签和食谱介绍。
* **底层字体 (The Reality):** 冰冷的等宽字体或无衬线系统字体（如 *Courier New*, *Roboto Mono*），用于 Page 3 的闪现条款、Page 5 的代码片段和警告弹窗。

## 6. 技术实现说明 (Technical Implementation Notes)

### 6.1 前端框架与状态管理
* **推荐栈:** React + Framer Motion (用于复杂的布局过渡与拖拽逻辑)，或纯 Vanilla JS + GSAP。
* **状态打分器:**
    * 维护一个简单的状态对象: `{ Exposure: 0, Control: 0, Seduction: 0 }`。
    * 在 Page 2 的 4 个选择完成后更新总分。
    * 在进入 Page 5 时，执行纯函数匹配优先级逻辑（最高分判定或组合判定），决定播放 `video_01_glossy.mp4` 到 `video_06_exploded.mp4` 中的哪一个。

### 6.2 媒体预加载优化 (Preloading Strategy)
* 这是保障沉浸感最核心的技术点。不能让用户在 Page 4 点击“Bake”后看到浏览器的 Loading 圈。
* **策略:** 在用户停留在 Page 1 和 Page 2 时，使用 `new Image()` 和 `<link rel="preload" as="video">` 异步静默加载核心音视频资产，确保进入 Page 4 和 Page 5 时的延迟低于 50ms。

### 6.3 界面文化（Interface Culture）的底层结合
在技术实现上，通过刻意设计极其平滑的贝塞尔曲线动画（Easing curves）和细腻的 CSS 阴影过度来构建 Page 2 的“完美操作感”。在 Page 5 渲染时，前端抛弃所有过渡动画，直接进行粗暴的 DOM 替换与元素强切，这种**前端实现手法上的落差**本身，就是对界面文化中“顺滑等同于掩盖”的最佳技术注脚。
